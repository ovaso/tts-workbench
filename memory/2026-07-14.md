# 2026-07-14

## 独立实时仿真页面接入

### 提问和更改方向

用户要求参考 `~/Projects/simulate-fs-rrs-proxy`，把 FreeSWITCH 调用 `aicc-tts-proxy` 的实时仿真能力搬到 Workbench。页面不能照搬原项目布局，需要使用 Vuetify 和 Material Design 风格，并作为侧边栏中的独立页面存在；本次明确不修改 `packages/core` 契约。

### 项目记忆

#### 当前项目状况和进展

- Web 左侧导航已新增“实时仿真”，路由为 `/realtime-simulator`。
- API 已新增独立 WebSocket 桥接端点 `/v1/realtime-simulator/bridge`。
- 仿真模块不注册 provider，不调用 TTS Facade，不生成 plan、mapping report 或 run archive，也不依赖 `@tts-platform/core`。
- 页面支持豆包、ElevenLabs、MyVocal、Lightning、MiniMax、CosyVoice 的代理模型路由配置。

#### 当前步骤所做内容

- 参考外部项目保留 `start_session → text_task_request → sentence_end` 协议流程。
- 支持逐行发送文本并保留中间和末尾空行，可配置分块间隔。
- 支持浏览器到 Workbench API、Workbench API 到目标代理的 WebSocket 双向桥接。
- API Key 只存在于页面内存和当前上游握手中，桥接以 `Authorization: bearer <API_KEY>` 发送，不写入日志或文件。
- 支持 PCM S16LE 单声道实时播放、停止本地播放、缓存音频和导出 WAV。
- 支持 WS 连接、会话启动、首包延迟、合成耗时、音频大小、音频时长和 RTF 指标。
- 使用 Vuetify 卡片、指标卡、状态 Chip、响应式双栏控制台和 Material Design 图标重新设计页面，没有沿用外部项目的暗色终端布局。
- 新增 API 桥接测试、Web 协议/音频工具测试和侧边导航测试。

#### 缺陷和风险点

- 实时仿真是面向本机调试的独立工具，浏览器 Origin 默认只允许 `localhost`、`127.0.0.1` 和 `::1`，避免任意外部站点滥用本机桥接。
- 目标代理地址仍允许任意 `ws://` 或 `wss://`，使用者需要确认目标可信；该能力不应直接暴露为公网服务。
- “停止本地播放”只清空浏览器播放队列，不会向代理发送 `interrupt_task_request`。
- WAV 返回模式只缓存和导出，不进行流式播放，也不估算裸 PCM 时长与 RTF。
- 当前环境的 Codex 浏览器插件在初始化时出现 `Cannot redefine property: process` 冲突，未能完成页面截图式视觉验收；全量类型检查、单元测试、生产构建、HTTP 路由访问和 WebSocket 错误响应验证均已通过。

#### 下一步内容或计划内容

- 在本机启动真实 `aicc-tts-proxy` 后，从实时仿真页依次验证豆包、MiniMax 和 CosyVoice 的完整握手、音频播放及 WAV 导出。
- 如果代理后续支持中断事件，可在独立仿真协议模块中补充 `interrupt_task_request`，不要把该协议加入 core canonical contract。
- 如需保存仿真记录，应建立仿真专用 archive，继续与标准 TTS Facade run archive 分离。

#### 推荐内容

- 保持实时仿真页定位为代理协议诊断工具，不把它并入 Benchmark/Arena 的可信执行链路。
- 真实验收时重点观察 `session_started`、首个二进制音频帧和 `tts_sentence_end` 三个阶段。
- API Key 调试完成后刷新页面清空内存，不要把密钥写入默认值、URL、Memory 或测试 fixture。

## 实时仿真页面布局调整

### 提问和更改方向

用户要求把“协议时间线”移动到页面下方，并让“连接与模型”和“文本流”在桌面布局中并排展示。

### 项目记忆

#### 当前项目状况和进展

- 实时仿真页桌面主操作区已改为 6/6 等分的双列布局。
- “协议时间线”已脱离右侧栏，移动到主操作区下方并占满 12 列。
- 窄屏下仍由 Vuetify 栅格自动切换为纵向排列。

#### 当前步骤所做内容

- 为“连接与模型”和“文本流”增加等高卡片容器。
- 文本流卡片改为纵向 flex，正文自动填充剩余空间，操作按钮保持在卡片底部。
- 移除时间线的 sticky 定位和视口高度绑定，改为下方全宽、内部限高滚动。
- 新增布局配置方法和单元测试，锁定连接卡片 6 列、文本卡片 6 列、时间线 12 列。

#### 缺陷和风险点

- 两张主卡片内容量差异较大，桌面端会以内容较高的“连接与模型”卡片作为整行高度基准；文本流按钮因此保持贴近卡片底部。
- Codex 浏览器插件仍受 `Cannot redefine property: process` 初始化冲突影响，本次未能生成自动截图。

#### 下一步内容或计划内容

- 在浏览器插件恢复后补充桌面和移动端截图式布局验收。
- 若后续希望进一步压缩首屏高度，可把“音频参数”改为可折叠区域，但当前不主动改变表单信息层级。

#### 推荐内容

- 保持协议时间线为全宽区域，便于查看较长 JSON 和连续音频事件。
- 桌面端优先使用 1280px 以上宽度进行代理调试，减少表单字段换行。
