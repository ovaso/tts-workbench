## Xiaomi MiMo V2.5 TTS 文档整理

### 提问和更改方向

用户要求按照其他厂商的文档整理归纳方式，整理 `docs/vendor_api_doc/xiaomi_mimo` 下的小米 MiMo TTS 文档。本次只做厂商资料沉淀，不实现 adapter。

### 当前项目状况和进展

- `docs/vendor_api_doc/xiaomi_mimo` 原本只有 `mimo_v2.5_tts.md`，内容仅包含官方文档链接。
- 官方 MiMo-V2.5-TTS 页面显示该系列使用 OpenAI Chat Completions 风格接口。
- MiMo-V2.5-TTS 当前包含 `mimo-v2.5-tts`、`mimo-v2.5-tts-voicedesign`、`mimo-v2.5-tts-voiceclone` 三个模型。

### 当前步骤所做内容

- 新增 `docs/vendor_api_doc/xiaomi_mimo/README.md`，整理文档入口、配置方式、模型能力摘要和推荐接入顺序。
- 扩展 `docs/vendor_api_doc/xiaomi_mimo/mimo_v2.5_tts.md`，补充文档来源、provider 接入定位、模型矩阵、预置音色、HTTP contract、流式 contract、voice design、instant voice clone、canonical mapping、vendor extension schema 建议和 capability 建议。
- 明确 `mimo-v2.5-tts-voiceclone` 更适合映射到 `voice.clone.instant`，不应作为持久 `voice.clone.create`。

### 缺陷和风险点

- 官方页面示例只明确非流式 `wav` 和流式 `pcm16`，输出格式集合仍需要 API 参考或真实接口验收确认。
- `mimo-v2.5-tts-voicedesign` 和 `mimo-v2.5-tts-voiceclone` 的流式目前是兼容式一次返回，不应在 capability 中夸大为低延迟实时能力。
- 即时音色复刻会把参考音频编码进 vendor request，真实 run archive 可能包含私密声音样本。

### 下一步内容或计划内容

- 若后续接入 `xiaomi_mimo` adapter，优先实现 `mimo-v2.5-tts` 的 `tts.sync` 和 `tts.stream`。
- 为 adapter 增加 capabilities、extension schema、examples、Contract.md 和 plan/mapping tests。
- 使用 injectable fetch/stream fixture 锁定 Chat Completions 非流式响应和流式 `delta.audio.data` 解析。

### 推荐内容

- 第一版公平 Benchmark 使用 `mimo-v2.5-tts`、固定预置音色、固定 `wav` 或流式 `pcm16`，避免把复杂自然语言风格控制混入 canonical 字段。
- Voice design 的音色描述应走 vendor extension，不新增 canonical 字段。
- Instant voice clone 应要求 consent，并在 archive 风险说明中提示不要提交真实样本。

## Xiaomi MiMo Adapter 接入

### 提问和更改方向

用户要求开始接入 Xiaomi MiMo 的 TTS 和音色克隆。本次按真实厂商接入底线实现 adapter、capabilities、extension schema、examples、Contract 文档和测试，并补齐平台对 `voice.clone.instant` 音频结果的 archive 支撑。

### 当前项目状况和进展

- Xiaomi MiMo provider 已注册到 API adapter registry，providerId 为 `xiaomi_mimo`。
- `mimo-v2.5-tts` 已支持 `tts.sync` 和 `tts.stream`。
- `mimo-v2.5-tts-voicedesign` 可通过 vendor extension 的 `voiceDesignPrompt` 走 `tts.sync`。
- `mimo-v2.5-tts-voiceclone` 已映射到 `voice.clone.instant`，不写入持久 voice registry。

### 当前步骤所做内容

- 扩展 core 契约：新增 `VoiceCloneInstantProviderResult`、`VoiceCloneInstantResult`，并让 `createInstantVoiceClone(plan)` 返回即时复刻音频结果。
- API facade 新增 `createInstantVoiceClone()`；路由新增 `POST /v1/voice-clones/instant`。
- `FileRunArchive` 新增 `writeVoiceCloneInstant()`，保存 request、plan、mapping report、vendor request、vendor response、result 和 audio。
- 新增 `apps/api/src/adapters/xiaomi-mimo/adapter.ts`、`capabilities.ts`、`extension-schema.ts`。
- 新增 `apps/api/src/adapters/xiaomi-mimo/Contract.md` 和 examples。
- `.env.example` 新增 `MIMO_API_KEY`。
- 新增 Xiaomi MiMo adapter 测试、instant clone archive 测试、API provider/route 测试，并更新 core contract 测试。

### 缺陷和风险点

- 尚未跑真实 Xiaomi MiMo 接口验收；当前通过 injectable fetch 和 SSE fixture 验证契约、映射和解析逻辑。
- MiMo 官方示例只确认非流式 `wav` 与流式 `pcm16`，当前 adapter 因此只声明这些输出能力。
- 即时音色复刻 archive 会把参考音频 data URI 写入 `vendor-request.json`，真实私密声音样本 run 目录不要提交。
- Web 音色管理页当前只适合持久复刻；MiMo instant clone 需要后续单独做“一次性复刻合成”入口。

### 下一步内容或计划内容

- 配置真实 `MIMO_API_KEY` 后，手动验收 `/v1/tts/sync`、`/v1/tts/stream` 和 `/v1/voice-clones/instant`。
- 根据真实返回补充错误码、usage 字段和输出格式 fixture。
- 如需前端支持，新增 instant clone 专用表单，不把即时复刻结果登记为持久音色。

### 推荐内容

- 公平 Benchmark 优先使用 `mimo-v2.5-tts`、固定预置音色和固定输出格式。
- Voice design 和风格控制继续留在 vendor extension 中。
- 真实验收后优先检查 `mapping-report.json`、`vendor-request.json`、`vendor-response.json` 和生成音频大小。

### 当前项目状况和进展

- API 已注册 `xiaomi_mimo` provider。
- 本地 `.env` 中的 `MIMO_API_KEY` 可被 API 进程读取。
- 已生成一段 `/tmp/mimo-reference.wav` 作为即时音色复刻测试参考音频。

### 当前步骤所做内容

- 启动本地 API：`http://127.0.0.1:4107`。
- 真实调用 `/v1/tts/sync` 成功，生成 run：`run_20260708070621_a5202bec`，音频 `audio.wav`，169004 bytes。
- 真实调用 `/v1/tts/stream` 成功，生成 run：`run_20260708070721_a09fbde8`，音频 `audio.pcm`，115200 bytes。
- 真实调用 `/v1/voice-clones/instant` 成功，生成 run：`run_20260708070734_c8ea14f6`，音频 `audio.wav`，145964 bytes。
- 检查即时复刻 mapping report，确认 `text`、`referenceAudio[0]`、`model` 均记录为 applied canonical fields，并保留参考音频 data URI archive 风险 warning。

### 缺陷和风险点

- 本次即时复刻参考音频是合成正弦波，不是真人语音，只能验证接口、归档和执行链路，不代表真实音色质量。

- 流式归档当前保存为 `audio.pcm`，浏览器播放器未必能直接播放，需要后续如有需要再加 PCM 到 WAV 包装或播放支持。

- 使用真实授权人声样本重新验收 `voice.clone.instant` 的音色效果。

### 推荐内容

- 后续改 MiMo adapter 时至少回归这三条链路：sync、stream、instant clone。
